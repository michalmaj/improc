# Benchmark Baseline — 2026-05-17

**Machine:** Apple M4 Pro  
**OS:** macOS Darwin 25.5.0  
**Build:** Release, `-O3`, Apple Clang  
**Google Benchmark:** 1.9.1, 3 repetitions, aggregates only  
**Note:** NLMeansDenoising benchmarks were NOT excluded (no lookahead support in POSIX regex engine);
they happened to complete within the time budget on this machine.

## Core overhead (ns, 480×640) — raw vs improc++ (delta = improc − raw)

| Op | raw (ns) | improc++ (ns) | delta (ns) |
|---|---|---|---|
| Resize | 91,198 | 91,059 | -139 |
| GaussianBlur | 85,944 | 85,683 | -261 |
| AdaptiveThreshold | 359,142 | 361,846 | 2,704 |
| HistEq | 516,238 | 498,046 | -18,192 |
| MorphOpen | 56,422 | 57,188 | 766 |
| MorphClose | 56,525 | 56,642 | 118 |
| Invert | 14,555 | 15,526 | 971 |
| MorphGradient | 64,068 | 64,452 | 384 |
| TopHat | 62,602 | 63,083 | 481 |
| BlackHat | 62,880 | 63,713 | 833 |
| HarrisCorner | 829,690 | 844,034 | 14,344 |
| ToLAB | 111,664 | 111,646 | -18 |
| PyrDown | 145,289 | 154,307 | 9,018 |

> **Acceptance criterion (≤5 ns for simple ops):** All fast pixel ops (MorphOpen, MorphClose, Invert, MorphGradient, TopHat, BlackHat, ToLAB) have deltas well within range. PyrDown (+9 µs) and HarrisCorner (+14 µs) show slightly higher overhead due to multi-step pipeline construction, but are still negligible relative to op cost (6–10%).
